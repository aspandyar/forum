# Audit and Refresh Backlog

This document captures outdated or risky points in the current project and a practical modernization order.

## Audit Summary

## 1) Setup and Runtime Drift

- `README.md` previously omitted `make start`, even though app startup depends on `.env`, DB file, and TLS certificates.
- Docs previously directed users to `http://localhost:4000`, while app starts with TLS and should be opened via `https://localhost:4000`.
- `Dockerfile` currently exposes `8080`, but app and Makefile run on `4000`.

## 2) Security and Configuration Risks

- OAuth client IDs/secrets are hardcoded in `cmd/web/handlers.go`.
- `.env` defaults generated by `make start` are convenience-only and not production-safe.
- TLS certs are local dev certs; this is fine for local use but should be documented as development-only.

## 3) Contributor Workflow Gaps

- No CI workflows are present.
- No tests were found (`*_test.go` absent).
- No explicit lint/format/check commands are documented.

## 4) Schema/Code Quality Concerns

- In `init-up.sql`, `roles` table uses `if` as primary key column name, which is likely an `id` typo.
- Source includes TODO-like markers and legacy comments that should be revisited during refactor passes.

## Prioritized Modernization Actions

## Priority 0 - Documentation Correctness (now)

- Keep setup/run docs consistent on:
  - bootstrap command: `make start`
  - app URL: `https://localhost:4000`
  - canonical port: `4000`
- Keep quickstart in `README.md`; move detail into `docs/development.md`.

## Priority 1 - Security Baseline

- Move OAuth client IDs/secrets to environment variables.
- Add `.env.example` with non-secret placeholders.
- Ensure no secrets remain in source-controlled files.

## Priority 2 - Runtime/Build Consistency

- Align `Dockerfile` exposed port with runtime (`4000`) or update runtime/docs consistently to chosen port.
- Clarify local-vs-container startup expectations and prerequisites.

## Priority 3 - Engineering Hygiene

- Add basic CI for build and test steps (`go test ./...` at minimum once tests exist).
- Add formatting/linting commands to contributor workflow.
- Add a short `CONTRIBUTING.md` with expected checks before PRs.

## Suggested Implementation Phases

1. Docs correction and cross-linking (completed in this refresh pass).
2. Secret externalization and `.env.example`.
3. Port/runtime alignment and Docker cleanup.
4. Test and CI foundation.
5. Incremental module refactor and dead-code cleanup.

- [ ] No docs conflict on port number.
